# Tea Shop

<br />
Этот прототип интернет-магазина демонстрирует все ключевые аспекты работы полноценного коммерческого сайта: от проектирования интерфейса до взаимодействия с API. Проект создан как основа для дальнейшего расширения и интеграции с реальными данными. Адаптивный и отзывчивый интерфейс обеспечивает удобный пользовательский опыт на мобильных устройствах и планшетах. Собственный API позволяет тестировать и разрабатывать функционал магазина в условиях, приближенных к реальной работе с сервером.
<br />
Link: https://teaonlinestore.netlify.app/

## Technologies Used

- Typescript
- React
- Redux Toolkit (включая createAsyncThunk для асинхронных запросов)
- SCSS (Sass)
- React Router
- Custom React Hooks
- Static Data Source (JSON)

## Features

- Просмотр списка товаров с указанием названия, цены, изображения
- Сортировка результатов выдачи по:
  - Типу чая (чёрный/зелёный/травяной)
  - Названию
  - Цене (по возрастанию и убыванию)
  - Рейтингу (по возрастанию и убыванию)
- Возможность поиска чая по названию
- Пагинация для навигации по результатам выдачи
- Просмотр страницы каждого отдельного чая с описанием
- Возможность добавления чая в корзину со страницы выдачи
- Возможность добавления чая в корзину со страницы карточки товара
- Возможность выбора веса товара (100г/200г) при добавлении в корзину и расчёт стоимости чая в корзине в зависимости от веса
- Подсчёт стоимости товаров, добавленных в корзину
- Удаление товаров из корзины
- Кастомный хук для получения данных о конкретном чае с помощью его id
- Кастомный хук для добавления чая в корзину
- Для целей демонстрации создан статический JSON-файл, предоставляющий данные о товарах

## Purpose & Outcome

- Необходимо было обеспечить удобство использования интерфейса на мобильных устройствах и планшетах, а также протестировать отзывчивость макета на разных экранах.

Решение: использование SCSS для создания адаптивной верстки, которая позволила пользователям получить единый опыт на всех устройствах. Пользователи отметили плавный и интуитивно понятный интерфейс на мобильных платформах.

- Для демонстрации функционала было необходимо реализовать взаимодействие с внешним источником данных.

Решение: разработка собственного API на основе статического JSON-файла для имитации запросов к серверу. Это позволило оптимизировать разработку, сократить время отклика и протестировать функционал в условиях, приближенных к реальным.

- Требовалось реализовать управление состоянием для обработки асинхронных запросов и обеспечить плавную навигацию между страницами.

Решение: использование Redux Toolkit и React Router позволило улучшить структуру приложения, упростить управление данными и организовать навигацию по страницам.
